## 2026-07-15
* **Completion**: Implemented all 8 slices of the [TUI plan](/docs/knowledgebase/plans/tui-projects.md): (1) client layer, (2) navigation model, (3) status-line hints, (4) projects home + project detail, (5) agent context with live SSE, (6) invoke with SSE streaming, (7) cluster topology, (8) palette lifecycle commands + new-project form modal. The TUI is a pure client of the node API, uses bubbletea/lipgloss v2, and never imports internal/server. All slices pass `go test -race`, `golangci-lint` (0 issues), and `go build`.
* **Completion**: Landed [Phase 3.6 — AAP host](/docs/knowledgebase/plans/aap-host.md); marked it complete in the [roadmap](/docs/knowledgebase/plans/roadmap.md) and updated the [AAP concept doc](/docs/knowledgebase/concepts/agent-adapter-protocol.md). The node spawns an AAP adapter over the stdio binding, runs the `initialize`→`ready`→prompt→turn loop, and shuts it down gracefully (`internal/server/aaphost.go`). A second agent *kind* alongside native ADK is declared in config (`agents.<name>.kind: aap`), sharing the `agentProc` map, invoke API, and project assignment. AAP turns bridge to the invoke SSE stream with `Last-Event-ID` resume (`internal/server/aapinvoke.go`); `context`/`status`/`error`/`approval_request` frames light up the `applyStatus`/`applyContextUpdate`/`applyError`/`applyApprovalRequest` receivers for full-fidelity execution context. Approval policy is the `auto_approve` config knob (human approve/deny UI + decision endpoint, the websocket binding, a real external adapter, `resume_token`, and MCP provisioning are deferred follow-ups). The `horde aap-mock` fixture is the end-to-end integration target.
